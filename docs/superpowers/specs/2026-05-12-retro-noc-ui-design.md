# Retro NOC 管理台界面优化设计

## 背景

当前前端是 React + TypeScript + Ant Design 管理台，主布局在 `ProxyConfig`，各业务页以独立 CSS 覆盖为主。现有界面偏常规后台模板，缺少与“家庭代理 / NAS 网关 / 网管系统”匹配的辨识度。

本次设计目标是把管理台改造成“90 年代网管系统”风格的 Retro NOC 控制台，同时保留现代后台的可读性、响应速度和表单可用性。

## 设计目标

- 视觉气质：灰色工业面板、凸起边框、青绿色在线状态、等宽字体点缀、终端日志区、老式网管系统窗口感。
- 桌面端：作为完整管理入口，继续支持所有现有配置、查询、表单、表格和弹窗能力。
- 移动端：定位为轻运维状态面板，优先查看状态和执行少量高频操作。
- 一致性：通过统一主题变量和通用样式层收敛颜色、按钮、卡片、表格、状态标签，减少页面间风格割裂。
- 可用性：复古只体现在视觉语言，不牺牲文本可读性、点击区域、响应式布局和 Ant Design 交互惯例。

## 桌面端设计

桌面端采用 Retro NOC 控制台布局。左侧导航保留当前分组结构，但视觉上改为灰色凸起面板和深青色选中态；顶部栏改为系统状态条，展示当前用户、在线状态和页面上下文。

内容区使用统一复古容器样式：卡片为浅灰面板，边框模拟老式窗口凸起效果；关键状态使用青绿色、琥珀色、红色作为 OK、Warning、Error 语义色；图表区可加入暗底网格和终端风格标题，但图表本身仍保持清晰。

需要优先覆盖的页面是：

- 登录页：改为网管系统登录窗口，不再使用现代紫色渐变背景。
- 主布局 `ProxyConfig`：更新侧栏、顶部栏、内容背景、默认概览入口。
- 状态类页面：Dashboard、流量概览、WOL、磁盘监控。
- 轻配置页面：用户管理、入站配置、路由规则。

其他页面可继承主题层基础样式，不在第一阶段做逐页深度重绘。

## 移动端设计

移动端定位为掌上状态面板，不复制桌面端完整信息密度。首屏展示：

- 今日流量和基础趋势入口。
- 磁盘健康摘要。
- WOL 快捷操作。
- 关键状态或最近日志摘要。
- 入口宫格：流量概览、磁盘监控、用户管理、入站配置、路由规则。

导航在移动端使用顶部状态条 + 抽屉菜单，不保留常驻桌面侧栏。列表和表格页面优先使用卡片化摘要；复杂列过多时允许横向滚动兜底，但关键字段必须在首屏卡片中可见。

用户管理、入站配置、路由规则在手机上主要支持查看、搜索、状态识别和必要轻操作。复杂新增/编辑表单可以进入，但不作为移动端体验优化重点。

## 实现约束

- 不新增后端 API，不改变现有路由路径和权限逻辑。
- 保持 React、TypeScript、Ant Design 技术栈，不引入新的 UI 框架。
- 建立统一 CSS 变量和通用 class，再逐步覆盖具体页面，避免每页重复定义颜色和阴影。
- 复古字体使用系统等宽字体栈点缀，不依赖外部字体下载。
- 移动端断点以 `768px` 为主要切换点，窄屏按钮和表格必须避免文字溢出。
- 不使用大面积高饱和渐变、装饰性光效或难以长期阅读的低对比配色。

## 非目标

- 不把界面做成真正的 90 年代低可用网页。
- 不重构业务数据流、Redux、API client 或后端服务。
- 不在第一阶段重绘所有次要页面。
- 不新增暗色主题切换功能；Retro NOC 是本次默认主题方向。

## 验收标准

- 桌面端 `/config/dashboard`、流量概览、WOL、磁盘监控、用户管理、入站配置、路由规则视觉风格一致。
- 移动端可以清晰访问 WOL、流量概览、磁盘监控、用户管理、入站配置、路由规则。
- 移动端无主要文本重叠、按钮文字溢出、内容被侧栏遮挡等问题。
- 登录页与管理台视觉风格一致。
- 前端构建通过，现有前端测试通过或明确记录与本次视觉改造无关的失败。
- 浏览器人工检查覆盖桌面宽屏和移动窄屏两个视口。
